## 一、团队成员与角色分工

1. 盛浩宇—— 需求负责人
2. 缪宇轩—— 架构负责人
3. 缪周洋—— 开发负责人
4. 朱雨乐—— 测试负责人

### 各角色职责

#### 需求负责人

1. 负责需求调研、访谈记录、需求文档撰写
2. 整理裁判文书标注规则、业务流程
3. 组织需求评审，维护需求变更记录
4. 对接 AI 工具生成需求初稿并审核

#### 架构负责人

1. 负责技术选型、系统架构设计
2. 设计数据结构、图示模块、标注存储方案
3. 搭建 Git 仓库、CI 环境
4. 编写架构决策记录（ADR）

#### 开发负责人

1. 制定编码规范、代码审查规则
2. 负责核心功能开发：标注编辑器、关系嵌套、图示生成
3. 审核 AI 生成代码，确保符合规范
4. 协调开发进度，解决技术问题

#### 测试负责人

1. 设计测试用例、执行功能测试
2. 负责标注一致性、图示正确性测试
3. 记录 Bug，跟踪修复
4. 输出测试报告

## 三、团队工作制度（队员守则）

### 1. 会议制度

- 每周固定开会时间：**每周三 晚 20:00**
- 会议时长：**15–20 分钟**
- 会议形式：线下会议/线上语音 / 视频
- 会议内容：进度同步、问题反馈、任务分配

### 2. 汇报制度

- 每日简单进度：群内文字汇报
- 每周正式汇报：由阶段主导人整理
- 问题必须**当天上报**，不拖延、不隐瞒
- 所有决策、变更必须记录在文档

### 3. 工作流程

1. 任务从 Git Issues 领取
2. 开发在 feature 分支完成
3. 提交前必须自测
4. 代码需经过 Code Review
5. AI 生成内容必须人工审核并标注来源

### 4. 文档与记录规范

- 所有文档存放在 Git 的 docs/ 目录
- 文档命名：阶段 - 名称.md，如 P0 - 团队章程.md
- AI 交互 Prompt 必须完整记录
- 需求变更、问题、决策必须写入日志
- 每周提交一份阶段进展小结

## 四、AI 工具链选型（本项目专用）

### 1. AI 编程助手

候选：GitHub Copilot、通义灵码

选择：GitHub Copilot

理由：适配 Python/Django/Vue，提示稳定，团队熟悉。

### 2. AI 对话工具

候选：Claude 3、文心一言

选择：Claude 3

理由：长文本理解强，适合法律标注规则、需求文档。

### 3. AI 代码审查工具

候选：CodeRabbit、AI Code Review

选择：CodeRabbit

理由：自动检查风格、安全、逻辑问题。

### 4. AI 测试工具

候选：Testim AI、腾讯云 AI 测试

选择：Testim AI

理由：可生成界面测试用例，覆盖标注与图示模块。

## 五、沟通与协作纪律

1. 群消息 24 小时内必须回复
2. 不无故缺席会议，缺席需提前说明
3. 任务延期必须提前说明并调整计划

## 六、项目目标

完成**法律裁判文书论证结构标注系统**，实现：

1. 文书导入、自动提取裁判理由
2. 命题标注（4 类 + GM 子类型）
3. 关系标注（5 种，支持嵌套）
4. 自动生成论证图示
5. 多人协同标注与差异比对
6. 数据导出、任务管理、权限控制
