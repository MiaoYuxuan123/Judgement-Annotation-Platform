## 一、标注界面类 Bug

### Bug-001 命题序号自动排序问题

**问题现象**

删除中间命题后，后续命题序号不会自动前移补齐，出现断号、空号、序号错乱。

**复现步骤**

1. 原文中标注生成多条有序命题；

2. 删除其中任意一条中间命题；

3. 观察左侧命题列表序号变化。

   **根因分析**

   仅做单条数据删除，未触发全量命题按文本顺序重新排序、重编序号逻辑。

   **修复方案**

   删除命题后，后端 / 前端重新遍历所有有效命题，按原文先后顺序重新自动分配连续序号，自动前移补齐。

   **验证结果**

   任意位置删除命题，后续序号自动连续补齐，无断号、重复、跳号。

### Bug-002 标签选择时二级标签弹不出

**问题现象**

选中一级标签后，对应的二级标签下拉面板无法弹出，不能选择细分标签。

**复现步骤**

1. 选中一条命题点击编辑标签；

2. 选择 GM 等一级标签；

3. 等待二级标签弹窗加载。

   **根因分析**

   一级标签选中事件未绑定二级标签渲染逻辑，接口标签分类数据回显异常，前端弹窗渲染触发失效。

   **修复方案**

   修复一级标签联动事件，完善标签树形数据回显，修复二级标签弹窗渲染逻辑。

   **验证结果**

   选择一级标签后二级标签正常弹出，可正常选择、绑定细分标签。

### Bug-003 修改标签时无法点击确定保存

**问题现象**

修改命题标签后，确定按钮点击无响应，无法保存标签修改。

**复现步骤**

1. 进入已有标签的命题编辑；

2. 重新更换一级 / 二级标签；

3. 点击确定保存。

   **根因分析**

   确定按钮绑定的提交事件失效，表单校验未通过但无提示，数据提交接口未触发。

   **修复方案**

   修复按钮点击事件绑定，补齐表单校验逻辑，保证合法标签可正常提交入库。

   **验证结果**

   修改标签后点击确定可正常保存，标签信息实时更新到命题列表。

### Bug-004 已添加标签的命题无法修改、删除

**问题现象**

已经绑定标签的命题，无法再次编辑标签，也无法执行删除操作。

**复现步骤**

1. 新建命题并完成标签保存；

2. 尝试重新编辑标签或删除该命题。

   **根因分析**

   已标注命题被错误设置为只读锁定状态，未开放编辑和删除权限接口。

   **修复方案**

   调整权限逻辑，未到任务锁定阶段允许已标注命题重新改标签、删除重建。

   **验证结果**

   已打标签命题可正常编辑修改、可正常删除，功能不受限制。

### Bug-005 多元关系无法控制槽位个数

**问题现象**

创建多元关系时，无法自定义控制参与命题槽位数量，不能按需增减。

**复现步骤**

1. 进入关系标注面板；

2. 选择多元关系类型，尝试调整参与命题个数。

   **根因分析**

   多元关系槽位为固定写死数量，未做动态配置和手动调节入口。

   **修复方案**

   增加槽位动态增减控件，支持按需添加 / 减少参与命题槽位，适配多元关系标注。

   **验证结果**

   可自由控制多元关系槽位数量，能适配 2 个及以上命题组合标注。

### Bug-006 槽位添加无关系类型权限控制

**问题现象**

通过添加按钮可无限增加槽位，没有限制哪些关系允许多槽位、哪些只能二元。

**复现步骤**

1. 选择普通二元关系；

2. 连续点击添加槽位按钮，可无限新增。

   **根因分析**

   未对不同关系类型做槽位最大数量约束，前端无类型判断拦截。

   **修复方案**

   配置关系类型规则：二元关系限制最多 2 个槽位，多元关系允许自定义扩展，超出则禁用添加按钮。

   **验证结果**

   二元关系无法多余添加槽位，多元关系可正常扩展，规则约束生效。

### Bug-007 增加槽位后切换关系，出现多余槽位

**问题现象**

先给二元关系增加多余槽位，再切换回普通二元关系，界面仍保留 3 个及以上槽位，逻辑错乱。

**复现步骤**

1. 选中关系，手动新增多个槽位；

2. 直接切换为标准二元关系类型；

3. 查看槽位数量。

   **根因分析**

   切换关系类型时，未重置槽位为默认基数，保留了之前手动新增的多余槽位。

   **修复方案**

   切换关系类型时，自动重置槽位到该类型默认数量，清空额外手动添加槽位。

   **验证结果**

   切换关系后槽位自动回归标准数量，不会遗留多余无效槽位。

### Bug-008 论证图示生成严重异常，无关命题也显示

**问题现象**

没有建立任何关系的孤立命题，也被自动渲染到论证图示面板中，图示杂乱。

**复现步骤**

1. 只标注多条命题，不创建任何关系；

2. 打开右侧论证图示区查看。

   **根因分析**

   图示渲染逻辑遍历全部命题，未做「是否存在关联关系」过滤。

   **修复方案**

   图示只渲染存在关系关联的命题，孤立无关系命题不进入图示渲染。

   **验证结果**

   无关系的命题不再出现在图示，只展示有关联的命题与连线。

### Bug-009 多元关系连线拥挤、排布错乱

**问题现象**

多元关系多个命题连线扎堆、交叉重叠，节点布局混乱看不清逻辑。

**复现步骤**

1. 创建 3 个及以上命题组成多元支持关系；

2. 查看右侧论证图示布局。

   **根因分析**

   图示自动布局算法未适配多节点多元关系，节点排布无避让、无分层。

   **修复方案**

   优化多元关系节点自动分层布局，增加节点间距避让，连线自动分流不扎堆。

   **验证结果**

   多元关系节点排布规整，连线不拥挤、不大面积交叉，结构清晰。

### Bug-010 嵌套关系箭头指向错乱

**问题现象**

创建嵌套组合关系后，图示箭头指向错误、指向反向或飘移错位。

**复现步骤**

1. 先建立组合关系作为中间节点；

2. 再和其他命题建立嵌套关系；

3. 观察箭头指向。

   **根因分析**

   嵌套组合节点 ID 与普通命题 ID 渲染逻辑未区分，箭头锚点匹配错误。

   **修复方案**

   区分普通命题与嵌套组合节点的渲染锚点，修正箭头起止指向逻辑。

   **验证结果**

   嵌套关系箭头指向准确，逻辑关系和图示箭头完全对应。

### Bug-011 标注提交按钮点击无反应

**问题现象**

完成命题、标签、关系标注后，点击提交按钮无弹窗、无保存、无任何反馈。

**复现步骤**

1. 完成一整套标注操作；

2. 点击页面提交保存按钮。

   **根因分析**

   提交接口异常、前端请求未发起，或表单存在未校验空值导致提交静默失败。

   **修复方案**

   修复接口请求路径，完善前端表单非空校验，增加点击加载状态与失败提示。

   **验证结果**

   点击提交可正常保存标注数据，成功有提示，非法输入有校验弹窗。

### Bug-012 同一命题被多个命题支持时箭头重叠

**问题现象**

一个命题被多个其他命题同时支持，多条箭头完全重叠覆盖，无法分辨多条关系。

**复现步骤**

1. 创建命题 1、2、3 同时支持命题 4；

2. 查看图示连线样式。

   **根因分析**

   同终点多条连线采用同一路径绘制，无偏移避让逻辑。

   **修复方案**

   多条指向同一命题的箭头做微小角度偏移、分层排布，避免完全重叠。

   **验证结果**

   多条支持箭头错落分布，不重叠、可清晰分辨每一条关系。

## 二、裁定界面类 Bug

### Bug-013 裁定界面部分修改跳转空白标注页

**问题现象**

在裁定页点击修改某条标注差异时，跳转后进入空白标注界面，无原文、无命题数据。

**复现步骤**

1. 进入任务待裁定页面；

2. 点击对某条标注进行修改；

3. 观察跳转页面内容。

   **根因分析**

   跳转路由未携带当前文书 ID、任务 ID 参数，页面加载时无法拉取对应文书和标注数据。

   **修复方案**

   修复跳转路由参数传递，打开编辑页自动携带任务、文书、标注 ID，正常拉取数据渲染。

   **验证结果**

   从裁定页点击修改，可正常跳转到对应完整标注界面，数据加载完整无空白。

### Bug-014 保存裁定结果时篡改原始标注

**问题现象**

裁定者保存裁定版本时，会同步覆盖、修改两名标注员原本的原始标注记录。

**复现步骤**

1. 双人完成独立标注；

2. 裁定者编辑并保存最终版本；

3. 回看原始两人标注记录。

   **根因分析**

   存储逻辑错误，直接覆盖原始标注数据，未做「原始标注只读、裁定版本另存」隔离。

   **修复方案**

   隔离存储：保留两名标注员原始数据不可改动，裁定结果单独生成最终版本，互不覆盖。

   **验证结果**

   裁定保存只生成最终版，不修改两位标注员原始标注记录，数据隔离正常。

### Bug-015 导出压缩包内容不符合规范要求

**问题现象**

裁定完成后导出压缩包，内部文件格式、目录结构、缺少标签版本 / 图示文件，和需求规范不符。

**复现步骤**

1. 任务进入可导出阶段；

2. 打包导出压缩包，解压查看内容结构。

   **根因分析**

   导出打包脚本未按规范目录组织，未自动附带标签版本、图示、Excel/JSON 全套文件。

   **修复方案**

   按需求规范重构导出打包逻辑，固定目录结构，齐全包含标注数据、图示、版本说明文件。

   **验证结果**

   压缩包目录规整，文件齐全，符合项目导出格式要求。

### Bug-016 任务状态「标注中 — 待裁定 — 可导出」流转错误

**问题现象**

任务状态跳转错乱，未完成双人标注就可进入待裁定，未裁定完成可直接跳到可导出。

**复现步骤**

1. 创建任务只完成单人标注；

2. 查看任务状态是否可手动切换到下一阶段。

   **根因分析**

   缺少各阶段前置条件校验，状态流转无依赖约束，可随意手动切换。

   **修复方案**

   增加状态流转前置校验：必须双人标注完成才允许进入待裁定；裁定完成才开放可导出，流程不可逆。

   **验证结果**

   任务严格按「标注中→待裁定→可导出」顺序流转，不满足前置条件无法跳转。



## 三、任务创建者界面类 Bug

### Bug-017 任务角色默认界面错误套用超级管理员主页

**问题现象**

任务创建者、标注员、裁定者登录后，默认加载了超级管理员主界面，出现大量无关无用 UI 菜单和无效路由入口。

**复现步骤**

1. 使用任务创建者 / 标注员 / 裁定者账号登录系统；

2. 进入首页查看菜单、侧边栏及路由页面。

   **根因分析**

   前端路由守卫未按角色匹配专属首页，所有角色统一渲染了超管布局模板，未做角色布局隔离。

   **修复方案**

   新增角色路由分发逻辑，不同角色登录自动匹配专属布局与菜单，隐藏无关权限 UI 和路由。

   **验证结果**

   各角色登录只展示自身权限对应的菜单与页面，无多余超管 UI，路由访问权限隔离正常。

### Bug-018 任务 / 数据目录角色身份展示异常

**问题现象**

任务目录、数据目录列表中，仅能正常显示**标注员、裁定者**身份，**任务创建者**身份标识显示错乱、空白或不识别。

**复现步骤**

1. 以任务创建者身份创建并分配任务；

2. 进入任务目录、数据目录查看角色身份标签。

   **根因分析**

   前端角色枚举字典缺少**任务创建者**匹配项，接口返回角色值未做前端映射渲染。

   **修复方案**

   完善前端角色枚举配置，补全任务创建者身份映射，列表正常渲染身份标签。

   **验证结果**

   任务创建者身份可正常展示，与标注员、裁定者身份标识统一规范显示。

### Bug-019 任务列表操作按钮缺失

**问题现象**

任务列表仅显示「查看详情」按钮，**标注、裁定、查看结果、导出** 等权限按钮不显示，有权限用户也看不到入口。

**复现步骤**

1. 用对应权限账号进入任务列表；

2. 查看每条任务的操作按钮栏。

   **根因分析**

   前端按钮渲染权限判断逻辑缺失，未根据当前用户角色 + 任务阶段动态渲染对应操作按钮。

   **修复方案**

   重构操作按钮渲染规则，按**角色身份** + **任务当前状态**动态展示标注 / 裁定 / 查看结果 / 导出按钮。

   **验证结果**

   不同角色、不同任务阶段自动展示对应可用操作按钮，无权按钮自动隐藏。

### Bug-020 详情页查看指南版本后无法正常返回

**问题现象**

在任务详情展开页面点击查看标签指南版本后，跳转新页面，无法原路返回上一级详情界面。

**复现步骤**

1. 进入任务详情展开页；

2. 点击查看指南版本；

3. 尝试返回原详情页面。

   **根因分析**

   页面跳转采用新开标签页无返回路由栈，或返回按钮路由路径配置错误。

   查看指南版本后可正常关闭 / 原路返回，不脱离任务详情页面上下文。

### Bug-021 同一任务内同一文书异常重复展示

**问题现象**

同一个任务目录下，同一份裁判文书重复多条显示，造成列表冗余混乱。

**复现步骤**

1. 任务创建者上传同一份文书一次；

2. 进入任务文档列表查看展示条数。

   **验证结果**

   同一任务下同一份文书只展示一条，无重复冗余条目。

### Bug-022 同一账号多权限叠加时裁定界面加载失败

**问题现象**

   重构多角色权限兼容逻辑，裁定界面按当前任务身份而非全局角色判定，隔离权限冲突。

   **验证结果**

   账号拥有多重权限仍可正常进入裁定界面，数据加载正常，功能可用。

### Bug-023 修改任务配置可重复载入同一文件

**问题现象**

## 四、 超级管理员界面类 Bug

### Bug-025 文书总库上传文件后提取文本直接入库，不可修改原文

**问题现象**

超级管理员在文书总库上传裁判文书后，系统自动提取文本并直接入库，**没有编辑入口，无法人工修改、校正原文内容**。

**复现步骤**

1. 进入超级管理员 - 文书总库；

2. 上传 PDF/Word/ 文本文书；

3. 查看提取后的原文，尝试编辑修改。

   **根因分析**

   上传解析完成后直接落库，未预留**人工编辑校正**页面与保存接口，默认只读不可改。

   **修复方案**

   新增文书文本预览与编辑页面，解析入库前允许管理员手动修正文本内容，确认后再正式入库。

   **验证结果**

   上传文书后可在线编辑、修正提取文本，修改后正常保存入库，原文可人工校准。

### Bug-026 用户管理角色编辑无约束，可随意分配不合理权限

**问题现象**

超级管理员在用户管理编辑角色时，没有权限规则约束，可任意搭配、乱分配角色身份，不符合权责分离规范。

**复现步骤**

1. 进入超管用户管理界面；

2. 编辑任意用户角色；

3. 随意勾选多个冲突角色保存。

   **根因分析**

   角色编辑前端无互斥校验、后端无角色规则拦截，未限制冲突角色同时赋予同一用户。

   **修复方案**

   增加角色互斥与业务规则约束，限制不合理角色组合；前端做单选 / 互斥选择，后端二次校验拦截。

   **验证结果**

   只能分配合规角色组合，冲突角色无法同时勾选，角色分配规范受控。

### Bug-027 配置中心可无一级标签直接新增二级标签

**问题现象**

在标签配置中心，不选择、不关联任何一级标签，仍能独立添加二级标签，造成标签体系数据孤立、层级错乱。

**复现步骤**

1. 进入超级管理员配置中心 - 标签管理；

2. 直接新增二级标签，不归属任何一级标签；

3. 保存成功无任何提示拦截。

   **根因分析**

   二级标签新增未做**必须绑定一级标签**的前置校验，前后端均缺少层级关联约束。

   **修复方案**

   前端强制必须先选择一级标签才能新增二级标签；后端增加关联外键校验，禁止孤立二级标签入库。

   **验证结果**

   无归属一级标签时无法添加二级标签，标签层级结构完整规范。

### Bug-028 修改标签指南保存只能覆盖旧版本，无法自定义版本名

**问题现象**

超级管理员编辑已有标签指南后，保存只能直接覆盖原版本，**不能另存新版本、也无法修改版本名称**，无法做多版本管理。

**复现步骤**

1. 进入配置中心标签指南管理；

2. 编辑已存在的指南配置；

3. 点击保存，无版本命名、另存为选项。

   **根因分析**

   编辑逻辑固定为**原地覆盖更新**，未设计版本新增、版本重命名、版本归档功能。

   **修复方案**

   修改保存逻辑：支持「覆盖原版本」和「另存为新版本 + 自定义版本名称」两种模式，保留历史版本。

   **验证结果**

   可选择覆盖旧版，也可新建版本并自定义版本名，实现多版本标签指南管理。

### Bug-029 用户登录后在线状态无法自动刷新更新

**问题现象**

用户登录系统后，超管后台用户列表的**在线状态不实时变更**，仍显示离线，状态不同步。

**复现步骤**

1. 普通账号登录系统；

2. 切换到超级管理员用户管理列表；

3. 查看该用户在线状态。

   **根因分析**

   登录接口未同步更新在线状态字段，页面无轮询 / 心跳刷新机制，状态数据静态不更新。

   **修复方案**

   用户登录成功后端自动刷新在线状态；前端增加轻量状态轮询或页面刷新自动拉取最新在线状态。

   **验证结果**

   用户登录后，超管用户列表在线状态即时更新，离线 / 在线状态展示准确。
