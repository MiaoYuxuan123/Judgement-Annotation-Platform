## 一、文档目的

本文档面向课程助教与验收评审，说明如何在本机启动系统、使用预置账号与测试数据，并按推荐路径完整演示系统核心功能（P0）与主要增强功能（P1）。

## 二、系统简介

本系统面向法学研究中的**法律论证分析**场景，对裁判文书（尤其是「裁判理由」部分）进行结构化标注与可视化，支持：

1. **任务创建**：选择文书、配置指南版本、分配标注员与裁定者  
2. **独立标注**：多名标注员各自完成命题标注与关系标注  
3. **冲突裁定**：裁定者对比多份标注结果，全部采纳或部分修改  
4. **结果导出**：导出命题/关系 CSV、论证图示 PNG，支持 ZIP 打包  

系统采用**四阶段不可逆流程**：`标注中 → 待裁定 → 可导出`（任务与文书均按此状态推进）。

## 四、运行方式

### 4.1 启动 MySQL 并配置环境变量

后端通过环境变量连接数据库（Windows PowerShell 示例）：

```powershell
$env:DB_COMMON_USER = "你的MySQL用户名"
$env:DB_COMMON_PWD  = "你的MySQL密码"
```

默认连接地址：

```text
jdbc:mysql://localhost:3306/jap?createDatabaseIfNotExist=true&...
```

首次启动时，若 `jap` 库不存在会自动创建；`DatabaseInitializer` 会自动建表并导入演示种子数据。

### 4.2 启动后端

```bash
cd backend
mvn spring-boot:run
```

后端默认地址：`http://localhost:8080`

也可先打包再运行：

```bash
cd backend
mvn -DskipTests package
java -jar target/judgment-annotation-platform-0.0.1-SNAPSHOT.jar
```

### 4.3 启动前端

新开一个终端：

```bash
cd frontend
npm install
npm run dev
```

前端默认地址：`http://localhost:5173`

Vite 已将 `/api` 代理到 `http://localhost:8080`，无需额外配置跨域。

### 4.4 访问系统

浏览器打开 `http://localhost:5173`，进入登录页。

### 4.5 验证启动成功

1. 后端控制台无数据库连接报错  
2. 前端登录页可正常显示  
3. 使用 `admin / 123456` 登录后，左侧导航出现「文书总库」「配置中心」「用户管理」

### 9.3 标注员（`annotator1` / `annotator2`）

#### 9.3.1 任务与数据选择

1. 登录 `annotator1`  
2. 任务目录**仅显示被分配的任务**（不显示未参与任务）  
3. 进入任务 1001 → 数据列表  
4. 观察各文书的「当前身份」（标注员）、「状态」（标注中 / 待裁定 / 可导出）  
5. 对「标注中」文书点击「开始标注」或「继续标注」  

#### 9.3.2 标注工作台（`/annotate/:taskId/:dataId`）

**演示点：系统核心功能**

工作台分为四个区域：

```
┌─────────────┬──────────────────────────┬─────────────┐
│  命题列表    │      原文展示区            │             │
│  关系列表    ├──────────────────────────┤    逻辑图示   │
│             │      关系生成区            │             │
└─────────────┴──────────────────────────┴─────────────┘
```

**步骤 A：命题标注**

1. 在原文区用鼠标**拖选**一段文本  
2. 弹出标签浮层，选择一级标签（SF / GF / SM / GM）  
3. 若选择 GM，继续选择二级标签（GM-L / GM-I 等）  
4. 确认后生成命题，编号为 P1、P2、P3…  
5. 已标注文本在原文中**高亮显示**  
6. 与已有命题范围**重叠**的文本无法再次标注  

**步骤 B：命题编辑**

1. 鼠标悬停命题列表项，显示「修改 / 删除」  
2. 修改：重新选择标签或调整文本范围  
3. 删除：删除后序号自动前移补齐（无断号）  

**步骤 C：关系标注**

1. 在关系生成区选择关系类型按钮：S / A / J / M / I  
2. 在各成员槽位下拉选择命题（或已有关系）  
3. S、A、M 为二元关系，默认两个成员  
4. J（组合）、I（同一）支持多个成员，点击「＋」增加槽位  
5. 点击「确认」生成关系，编号 R1、R2、R3…  
6. 关系列表支持修改、删除；点击关系行可激活编辑  

**步骤 D：逻辑图示**

1. 完成命题与关系后，关系生成区右上角显示「图示待刷新」  
2. 点击「**生成图示**」按钮（图示不会自动刷新，需手动触发）  
3. 右侧逻辑图示区展示论证结构图  
4. 可点击「↗ 全屏预览」放大查看  
5. 点击「**绘制图示** / **编辑图示**」进入论证图全屏编辑器（路径：`/annotate/:taskId/:dataId/graph`），基于 Vue Flow 调整节点位置、关系连线与布局，保存后回写 `graphLayout`  

**步骤 E：暂存与提交**

1. 「暂存」：保存草稿，不推进任务状态  
2. 「提交」：正式提交标注结果；当所有标注员均提交后，文书进入「待裁定」  

**推荐演示文案（任务 1001 / 劳动争议文书）：**

| 选中文本 | 建议标签 |
|----------|----------|
| 劳动者与用人单位建立劳动关系后，双方均应遵守劳动合同约定 | GM → GM-L |
| 现有考勤记录、工资流水可以证明申请人在案涉期间持续提供劳动 | SF |
| 公司主张双方不存在劳动关系，但未提交充分反证 | SF |
| 本院不予采纳 | SM |

可建立关系：`S(P2, P4)`（事实支持结论）、`A(P3, P4)`（反对关系）等。

#### 9.3.3 双人独立标注

1. 退出 annotator1，登录 `annotator2`  
2. 对同一文书独立标注（标签、关系可与 annotator1 不同）  
3. 两人均提交后，文书状态变为「待裁定」  

#### 9.3.4 消息中心（P1）

**入口：** 创建者 / 参与者布局顶栏右侧铃铛图标（`MessageCenter` 组件）。

**演示点：**

1. 登录 `creator`、`annotator1` 或 `reviewer`，顶栏可见消息入口与未读数  
2. 登录 `admin` 时不展示消息中心  
3. 任务阶段变更、裁定退回等操作会写入 `/api/messages`；点击消息可跳转相关任务  
4. 支持标记已读、全部已读、删除已读

### 9.4 裁定者（`reviewer`）

#### 9.4.1 进入裁定界面

1. 登录 `reviewer`  
2. 进入任务 1003（劳动争议裁定演示）  
3. 数据列表中「待裁定」文书点击「**开始裁定**」  
4. 进入 `/review/:taskId`  

#### 9.4.2 对比标注结果（P0）

**界面布局：**

- 左侧：命题列表、关系列表  
- 中间：原文展示区（带标注高亮）、论证图示（可收起/展开）  
- 右侧：标注员列表（参与者一、参与者二、裁定结果）  

**演示操作：**

1. 顶部下拉切换不同文书  
2. 右侧点击不同标注员，对比命题数量、标签差异、关系差异  
3. 原文区与图示区随选中标注员同步切换  
4. 可指出任务 1003 中 P1 的标签差异：annotator1 为 GM，annotator2 为 GF  

#### 9.4.3 全部采纳（P0）

1. 选中某一标注员的结果  
2. 点击「**全部采纳**」  
3. 确认弹窗 → 系统将该标注员结果定为最终版  
4. 文书进入「可导出」，返回数据列表  

#### 9.4.4 部分修改（P0）

1. 选中某一标注员或已有裁定草稿  
2. 点击「**部分修改**」  
3. 跳转至标注工作台（裁定模式），基于选中结果继续编辑命题/关系  
4. 点击「提交」保存为**裁定草稿（待确认）**  
5. 返回裁定界面，右侧出现「裁定结果（待确认）」  
6. 检查草稿内容后点击「**确认**」→ 裁定生效；或「**取消**」→ 丢弃草稿  

#### 9.4.5 查看已完成裁定（任务 1002）

1. 进入任务 1002 → 数据列表  
2. 文书状态为「可导出」  
3. 可进入裁定界面查看「裁定结果」，或进入结果页导出

### 9.5 结果查看与导出

**可访问角色：** 任务创建者、裁定者、标注员（查看）；导出以创建者与裁定者为主。

#### 9.5.1 单篇结果查看（`/results/:taskId`）

1. 使用 `creator` 或 `reviewer` 登录  
2. 进入任务 1002 → 数据列表 →「查看结果/导出」  
3. 页面展示：  
   - 各标注员版本（命题数 / 关系数）  
   - 最终裁定结果  
   - 原文高亮、论证图示  
4. 切换右侧「结果来源」对比不同版本  

#### 9.5.2 单篇 ZIP 导出（P0）

1. 在结果页点击「**导出 ZIP**」  
2. 浏览器弹出保存对话框，选择保存位置  
3. ZIP 内容结构：

```text
{任务名}_{文书名}_{日期}.zip
└── {文书ID}_{文书标题}/
    ├── 导出说明.txt
    ├── 参与者一/
    │   ├── 命题列表.csv
    │   ├── 关系列表.csv
    │   └── 论证图示.png
    ├── 参与者二/
    │   └── ...
    └── 最终裁定结果/
        ├── 命题列表.csv
        ├── 关系列表.csv
        └── 论证图示.png
```

#### 9.5.3 批量 ZIP 导出（P1）

1. 在数据列表页（`/tasks/:id/data`）  
2. 勾选多篇「可导出」文书  
3. 点击「批量导出 ZIP（N）」  
4. 生成包含多篇文书子目录的单个 ZIP 文件

## 十、端到端完整业务流程演示

以下演示从零走完「创建 → 标注 → 裁定 → 导出」全链路。

### 阶段 0：准备（admin）

1. 登录 `admin`  
2. （可选）上传 1 篇新文书或手动新增  
3. （可选）在配置中心确认指南版本  
4. 退出登录  

### 阶段 1：创建任务（creator）

1. 登录 `creator`  
2. 新增任务，名称如「答辩现场演示任务」  
3. 选取 1–2 篇文书（建议选较短的「劳动争议仲裁审查裁定」）  
4. 分配 annotator1、annotator2 为标注员，reviewer 为裁定者  
5. 创建成功，进入数据列表确认状态为「标注中」  
6. 退出登录  

### 阶段 2：双人标注（annotator1 → annotator2）

**annotator1：**

1. 进入新任务 → 开始标注  
2. 至少标注 3 个命题、2 个关系  
3. 生成图示并提交  

**annotator2：**

1. 同一文书开始标注  
2. 故意使用不同标签或关系（制造裁定差异）  
3. 提交  

→ 两人提交后，文书状态变为「待裁定」

### 阶段 3：裁定（reviewer）

1. 登录 `reviewer`  
2. 进入任务 → 开始裁定  
3. 切换对比两名标注员结果  
4. 选择「部分修改」→ 在标注工作台微调 1 处标签 → 提交草稿  
5. 返回裁定界面 → 确认裁定  
→ 文书状态变为「可导出」

### 阶段 4：导出（creator 或 reviewer）

1. 数据列表 → 查看结果/导出  
2. 查看最终裁定结果的图示  
3. 导出 ZIP，打开本地文件验证 CSV 与 PNG  

### 阶段 5：收尾（可选）

1. `creator` 查看任务目录，确认任务状态已推进  
2. `admin` 查看用户在线状态变化

## 十三、常见问题与排障

### Q1：前端请求接口失败 / 登录无响应

- 确认后端已启动在 `http://localhost:8080`  
- 确认前端通过 `npm run dev` 启动（自带 API 代理）  
- 检查浏览器控制台 Network 面板中 `/api` 请求状态码  

### Q2：后端启动报数据库连接错误

- 确认 MySQL 服务已启动  
- 确认已设置 `DB_COMMON_USER` 和 `DB_COMMON_PWD` 环境变量  
- 确认本机 3306 端口可连接  

### Q3：登录成功但看不到预置任务

- 确认使用的是正确角色账号（参与者只能看到被分配的任务）  
- 若数据库曾被手动清空，需重置数据库后重启后端（见下节）  

### Q4：后端打包失败（jar 无法重命名）

- Windows 上可能是旧 Java 进程占用 jar 文件  
- 结束相关 Java 进程后重新执行 `mvn -DskipTests package`  

### Q5：图示不显示

- 需先添加至少 1 个命题  
- 必须点击关系生成区的「**生成图示**」按钮（不会自动刷新）  
- 若刚修改了命题/关系，注意「图示待刷新」提示  

### Q6：无法进入裁定 / 导出

- 裁定：需所有标注员均已**提交**，文书才进入「待裁定」  
- 导出：需裁定已**确认**，文书才进入「可导出」  

### Q7：导出时浏览器无反应

- 部分浏览器需允许「下载多个文件」权限  
- 导出使用「另存为」对话框，请留意是否被弹窗拦截

## 十四、重置演示数据

若演示过程中数据被改乱，可重置为初始种子状态：

**方法一：删除数据库后重启（推荐）**

```sql
DROP DATABASE IF EXISTS jap;
```

然后重启后端，`DatabaseInitializer` 会检测空库并重新执行 `data.sql`。

**方法二：仅清空业务表**

保留 `sys_user` 等表结构，手动 TRUNCATE 业务表后重启并导入 `data.sql`（适合高级用户）。

> 注意：当前 `spring.sql.init.mode=never`，种子数据由 `DatabaseInitializer` 在**首次建库**时导入；已有库不会重复导入演示数据。
