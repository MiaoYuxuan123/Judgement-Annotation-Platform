# 裁判文书论证标注系统 需求规格说明书（SRS）

**符合** **IEEE** **830‑1998 标准**

## 1 引言

### 1.1 目的

本文档依据 **IEEE** **830‑1998 标准**，完整定义**裁判文书论证标注系统**的功能需求、界面布局、标签体系、命题与关系标注、论证图示、四阶段标注流程、角色权限、冲突裁定、数据导出与非功能约束。

本文档用于系统开发、测试、评审、项目管理与最终验收，确保科研教学与法律论证挖掘场景可落地。

### 1.2 范围

#### 1.2.1 包含功能

文本导入、裁判理由智能提取、命题抽取与自动编号、三级标签体系、命题标注、关系标注、论证图示自动生成、多人独立标注、双标注结果并排展示、人工冲突裁定、任务分阶段管理、数据导出、版本与指南管理。

#### 1.2.2 不包含功能

OCR 图文识别、扫描件解析、AI 自动论证推理、外部法律数据库实时对接、音视频处理、在线协作实时编辑。

### 1.3 定义、缩写

- **SRS**：Software Requirements Specification
- **命题**：从裁判文书中抽取的具有独立语义的语句单元
- **一级标签**：SF/GF/SM/GM 四类命题性质判断
- **二级标签**：GM 下的法律、解释、合同、习惯、道德、其他规范
- **关系**：支持 S、反对 A、组合 J、匹配 M、同一 I
- **四阶段流程**：创建任务 → 标注 → 冲突解决 → 结果输出（不可逆）
- **裁定者**：单个标注项目的唯一终审人，一次裁定生效
- **序号自动前移**：删除某命题后，后续序号依次向上补齐

## 2 总体描述

### 2.1 产品前景

本系统面向法学研究者、教师、学生、律师、法务人员，用于**裁判文书论证结构结构化标注、可视化与数据集沉淀**，为论证挖掘、自动化推理模型训练、科研教学与律所办案提供标准化工具。

### 2.2 产品功能

系统主要功能包括：

1. 任务创建与管理
2. 裁判文书导入与文本提取
3. 命题与标签标注
4. 论证关系标注与图示生成
5. 多人协同标注与冲突裁定
6. 标注数据导出与版本管理

### 2.3 用户特征（权责分离）

1. **超级管理员**：用户管理、角色分配、权限配置、全局设置
2. **任务创建者**：上传文档、配置标签、分配标注员、指定裁定者
3. **标注员**：仅完成自身被分配的文本标注、命题标注、关系标注
4. **裁定者**：查看双标注结果、人工比对差异、编辑并确认最终版本

### 2.4 约束

- 仅处理纯文本，不处理图片、扫描件
- 任务启动后**标签体系不可修改**
- 命题序号**按文本出现顺序自动生成**，删除则后续自动前移
- 无撤销 / 重做，仅支持**删除→重新创建**
- 四阶段流程**不可逆**，导出后平台数据不可修改
- 单个标注项目**仅一名裁定者、一次裁定生效**
- 标注员不可查看他人中间标注，仅可查看最终结果
- 系统仅保留**最后一次提交**的标注结果

### 2.5 假设与依赖

- 输入以裁判文书为主，重点标注**裁判理由 / 本院认为**
- 用户具备基础法律文本阅读理解能力
- 运行环境为 PC 端主流浏览器
- 支持文件导入 / 导出、本地存储与下载

## 3 详细需求描述

### 3.1 对外接口需求

#### 3.1.1 用户界面

- 采用三栏布局：左侧原文 + 命题列表、中间标注区、右侧图示区
- 界面区域支持拖拽缩放，比例可自由调整
- 原文区高亮显示已标注命题，并显示序号
- 标注区提供标签选择面板与关系输入面板
- 图示区支持缩放、拖拽节点位置

#### 3.1.2 硬件接口

- 无特殊硬件接口，仅需 PC 及标准输入输出设备

#### 3.1.3 软件接口

- 支持 PDF、Word、纯文本文件导入
- 支持 Excel、JSON、PNG/JPG/SVG 文件导出

#### 3.1.4 通信接口

- 无特殊通信接口，基于标准 HTTP/HTTPS 协议通信

## 5 验收标准

1. 所有 **Must** 需求 100% 实现。
2. 完整支持四阶段流程：创建→标注→裁定→导出，**不可逆**。
3. 命题序号按文本顺序生成，删除后自动前移。
4. 双标注结果并排展示，裁定者一次终审生效。
5. 任务启动后标签体系锁定，不可修改。
6. 标注员只能查看自己标注与最终结果，不可看他人中间标注。
7. 导出 Excel/JSON/ 图片正常，平台数据不可修改。
8. 角色权限分离，由超级管理员统一分配。
